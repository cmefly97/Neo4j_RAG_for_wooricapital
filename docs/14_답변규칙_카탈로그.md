# 답변 규칙 카탈로그 (Source of Truth) — 2026-07-02

> 어제·오늘 프롬프트/답변을 다듬으며 생긴 규칙을 **한 곳에 정리**한 문서.
> 두 답변 경로(결정적 테이블 / LLM)가 이 카탈로그를 진실의 원천으로 삼아 파생된다.
>
> - 결정적 경로 룰 → `backend/app/search/answer_rules.py` (정책 플래그 + 템플릿)
> - LLM 경로 룰 → `backend/app/search/prompts.py` (프롬프트) + `cypher_qa.py`(컨텍스트 정책)
> - 규칙 변경 시 **이 문서와 해당 모듈을 함께** 갱신한다.

## 0. 경로 개요
질의는 라우터(`router.py`)로 분기된다.
- **결정적 테이블 경로**: 금리등급/네고/프로모션 등 수치형 → CSV 값 + 템플릿(LLM 무개입).
- **LLM 경로**: 서술형/그 외 → Hybrid 검색 + 프롬프트 기반 생성.

## 1. 결정적 테이블 경로 규칙 (`answer_rules.py`)

### 1-1. 동작 정책 플래그

| 플래그 | 기본값 | 의미 |
|---|---|---|
| `SHOW_BOTH_CUSTOMERS` | True | 고객유형 미지정 시 데이터 있는 유형(내국인·외국인) **모두** 출력. 명시 시 해당 유형만. |
| `LIST_TERMS` | True | 개월수 미지정 시 **전체 개월 금리 나열**(단일이면 값만). 지정 시 해당 구간 1건. |
| `NEGO_ARITHMETIC` | False | "최저금리"라도 `G/L − 네고` **산술 안 함**. G/L + 적용가능 네고 목록만 제시(상담사 판단). |
| `NEGO_FILTER_BY_QUERY` | True | 질문에 명시된 네고(거점장·증빙·HJ)만 표기. 미명시 시 전체. |
| `PROMO_REQUIRE_ALL` | True | 프로모션은 금리등급·NICE·KCB **모두 충족**해야 가능. |
| `SHOW_SOURCES` | True | 답변 끝에 출처 파일명 표기. |

### 1-2. 표현·판정 규칙
- **개월별 금리**: 여러 개월이면 "12개월 24.0%, 24개월 3.0%…"로 나열, 단일이면 값만.
- **고객유형별 네고율**: 네고율은 고객유형별로 다를 수 있어 CSV `customer_type`로 구분.
- **프로모션 가능/불가 판정**: 사용자가 준 값(NICE/KCB/등급)을 임계값과 대조 →
  미달 있으면 "아니요, 불가능합니다 …기준 미달", 모두 충족 "네, 가능합니다",
  일부만 제시 "충족하나 나머지 조건 확인 필요".
- **등급 비교 방향**: 등급은 숫자가 낮을수록 상위(`값 ≤ 기준`이면 충족).
- **개월 기본값**: 개월 지정 시 그 이하 최대 구간, 미지정 시(비나열 정책일 때) 최소 구간.

### 1-3. 데이터 출처
`backend/data/{rate_grade,nego_rule,promotion}.csv` — 스키마는 `backend/data/README.md`.
규칙 값이 바뀌면 CSV만 수정(서버 재시작 불필요, 요청마다 로드).

## 2. LLM 경로 규칙

### 2-1. 프롬프트 (`prompts.py`)

| 규칙 | 내용 |
|---|---|
| 역할 | "우리캐피탈 오토운영팀 상담을 돕는 도우미" |
| 근거 한정 | 제공된 '자료'만 근거로, 없으면 지어내지 말 것 |
| 미근거 처리 | (counselor) "규정에 명시되어 있지 않습니다" |
| 출처 | 답변 끝에 출처 파일명 명시 |
| 톤 | 상담사 톤·존댓말·완결 문장(예: "…차등 적용됩니다") |

### 2-2. 답변 스타일(UI 선택, `answer_mode`)

| 모드 | 지시 |
|---|---|
| `standard`(기본) | 정확하게 답변 |
| `concise` | 핵심만 1~2문장 |
| `detailed` | 조건·수치·등급·예외 빠짐없이 상세히(존댓말) |
| `counselor` | 상담 보조원 — 근거 기반, 미근거 시 "규정에 명시되어 있지 않습니다" |

### 2-3. 컨텍스트 구성 정책 (`cypher_qa.py`)

| 규칙 | 내용 | 근거 |
|---|---|---|
| 원문 1차·구조화 보조 | `[문서 발췌]`(청크) 먼저, `[구조화 사실]`은 보조로 뒤에("상충 시 발췌 우선") | 09 교훈 §4 |
| 입력 컨텍스트 캡 | 청크·총량 상한으로 입력 길이 제한(사내 모델 긴 입력 시 빈응답 방지) | 09 교훈 §5 |
| 저점수 fact 컷 | 구조화 사실 `score ≥ 3` 상위 N만 사용 | 08 §6 |
| 의도 우선 etype | `_intent_etypes`로 질문 의도 맞는 etype 가산 | 08 §5 |
| 빈응답 폴백 | 빈 답이면 thinking-off 재시도 → 그래도 없으면 발췌 폴백 | 09 교훈 §5 |

## 3. 라우팅 규칙 (`router.py`)

| route | 조건 | 처리 |
|---|---|---|
| `numeric` | 수치 힌트만(금리·등급·개월·%·네고·프로모션·숫자) | 결정적 테이블 우선 |
| `mixed` | 수치 + 규칙 힌트(가능·까지·판정·대상·여부·취급) | 테이블 시도 후 Hybrid+LLM |
| `descriptive` | 수치 힌트 없음 | Hybrid+LLM |

> 주의: "점"은 "점검/점포" 오매칭 방지로 힌트에서 제외(점수는 숫자로 이미 포착).

## 4. 유지보수 원칙
- 새 규칙은 **이 문서에 먼저 한 줄** 추가 → 해당 모듈(`answer_rules.py`/`prompts.py`)에 반영.
- 결정적 경로 문구·정책은 코드 하드코딩 금지, 반드시 `answer_rules.py` 경유.
- 값(금리·네고·프로모션 임계값)은 CSV, **규칙(표현·정책)은 모듈**, **근거·이력은 이 문서**.
