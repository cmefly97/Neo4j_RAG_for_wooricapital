## 오늘 한 일

### 1) 런타임 검색 로깅 추가
- `hybrid.retrieve` — `ACTION RETRIEVE`(벡터/풀텍스트 건수·상위 출처·lucene), `ACTION CTX_CHUNK`(융합된 최종 청크 하나씩).
- `_gen_answer` — `LLM_CALL`(model·mode·ctx_chars) / `LLM_ANSWER`(chars·preview) / `LLM_ERROR|EMPTY|RETRY`.
- 날짜별 로그(`logs/YYYY-MM-DD.log`)에 "벡터+풀텍스트 융합 → 최종 LLM 호출" 전 과정이 매 요청 기록됨.

### 2) 검색 정밀도 — RRF 유실 방지
- 증상: "나이스 885점, 2등급 최저금리"에서 **벡터는 답하는데 Hybrid는 못 찾음**.
- 원인: RRF가 벡터만 강한 정답 청크를 top-k 밖으로 밀어냄.
- 해결: **top-k 6→8** + **벡터 상위 3 보장**(RRF와 별개로 최종 세트에 우선 포함).

### 3) LLM 빈 응답 — 원인 규명 & 해결
- 로그로 확인: `6.9k자→정상 / 8.2k자→빈응답`이 thinking on/off·max_tokens 4096/8192 **모두 동일**.
- 결론: 출력 토큰이 아니라 **입력 컨텍스트 길이 한도 초과**가 원인(사내 모델).
- 해결: `_build_context`에 **총량 캡(≤5500자)** + 빈응답 시 **thinking-off 재시도** + **발췌 폴백**.
- 청크당 길이는 900→**1500 원복**(답변이 짧아지지 않게), 총량만 캡.

### 4) 프롬프트 정리 + 답변 스타일 옵션
- 프롬프트를 단순화: **"너는 우리캐피탈 오토운영팀 상담을 돕는 도우미다. 정확하게 답변해줘. / 자료만 근거,
  없으면 지어내지 말 것, 출처 명시."** (발췌 우선은 컨텍스트 라벨에 유지)
- **답변 스타일 3종 옵션**(UI 드롭다운 "✍️ 답변 스타일"):
  - `concise`(간단히) · `standard`(기본) · `detailed`(자세히·상담사 톤)
  - `/search`·`/search/compare`에 `answer_mode` 파라미터, `_gen_answer(answer_mode)`로 반영. 미지정=기본.

## 오후 추가 작업 — 의도 라우팅 + 결정적 답변 + 룰 분리

### 6) 질의 의도 라우팅 도입 (Phase 1)
- 배경: 모든 질문이 동일 Hybrid+LLM 경로 → 수치형(금리·프로모션)에서 빈응답·환각·오답 반복.
- `app/search/router.py` 신설 — `route_intent()`가 `NUMERIC_HINTS`/`RULE_HINTS`로
  **numeric / mixed / descriptive** 3분류. LLM 없이 결정적.
- `cypher_qa.search()` 통합: `ROUTE` 로깅 → 결정적 테이블 short-circuit → 없으면 기존 경로.
- 힌트 정밀화: "점"은 "점검/점포" 오매칭이라 제외(점수는 숫자로 포착). 참고 문서 `../12`.

### 7) 결정적 테이블 답변 경로 (선결사항)
- 수치형은 깨진 PDF 표를 LLM에 맡기지 않고 **큐레이션 CSV 값 → 템플릿 조립(LLM 무개입)**.
  → 빈응답·환각 원천 제거, 값이 결정적.
- `backend/data/{rate_grade,nego_rule,promotion}.csv` + `app/search/tables.py`.
- 확장 규칙 3종:
  1. **고객유형 미지정 시 내국인·외국인 모두** 출력(명시 시 해당 유형만).
  2. **개월별 금리 나열**(외국인 12/24/36/48개월 등), 단일이면 값만.
  3. **네고율 고객유형별 구분** + 질문에 명시된 네고(거점장·증빙·HJ)만 표기.
- 도메인 규칙: "최저금리"라도 `G/L − 네고` 산술 안 함(네고 목록 제시), 개월 미지정 시 최소 구간.

### 8) 프로모션 가능/불가 판정
- `promotion.csv`를 임계값 컬럼(`grade_req/nice_min/kcb_min`)으로 구조화.
- 사용자가 준 값(NICE/KCB/등급)을 파싱해 기준과 대조 → **가능/불가/확인필요** 판정.
  예) "나이스 955, KCB 800 프로모션 가능?" → "아니요, 불가능… KCB 800점(기준 976점) 미달".

### 9) 규칙 외부화 + 단일 카탈로그
- `app/search/answer_rules.py` — 결정적 경로 **정책 플래그 + 문장 템플릿**(로직과 분리).
- `app/search/prompts.py` — LLM 프롬프트/스타일을 `cypher_qa`에서 분리.
- `../14_답변규칙_카탈로그.md` — 두 경로의 모든 규칙을 한 문서(진실의 원천)로 정리.
- 계획 문서 `../13_의도라우팅_개선계획.md`(선결/Phase1~3).
