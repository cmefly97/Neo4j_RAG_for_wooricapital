### 2) 검색 파이프라인 진단
- `model=qwen` 검색이 `[문서 발췌]` 폴백으로 나옴 → 로그 분석 결과 **LLM 전부 사용 불가**였음:
  - Qwen(`223.130.140.218:8000`): **502 Bad Gateway**(nginx 뒤 모델 서버 다운)
  - Claude: **401 Authentication**(키 무효)
  - HCX-30B: 사내 게이트웨이라 현재 망에서 도달 불가
- 임베딩 검색(리트리브)은 정상 → 폴백은 "LLM 없을 때 근거 문서 노출"이라는 설계된 동작.

### 3) 다중 모델 선택 기능 제거 → HCX-30B-Text 단일 운영

  제각각이라 관리가 어려웠다.** 실제로 오늘도 Qwen 502 / Claude 401로 셋 다 죽어 있었음.
  → 기준 베이스 모델 **HCX-30B-Text(hcx-agent-05)** 하나만 남기고 나머지는 삭제.
- 정리 범위:

  - `app/config.py` — anthropic/hcx/qwen 설정 및 임베딩의 HCX 폴백 제거(embed_base/embed_key 단순화).
  - `.env` — ANTHROPIC/HCX/QWEN 블록 삭제(실키 포함), `EMBED_API_KEY=noauth`(서버 무인증, 클라가 키 요구).
  - `frontend/src/tabs/UserTab.tsx` — 모델 선택 드롭다운 제거 → 고정 표시(HCX-30B-Text). "다른 모델 선택" 안내문구 정리.
