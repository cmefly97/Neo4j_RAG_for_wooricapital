# 교훈 & 개선사항 (Lessons Learned) — GraphRAG PoC

> 다음 프로젝트에서 같은 실수를 반복하지 않기 위한 기록.
> 형식: **증상(무엇이 보였나) → 원인 → 해결/재발방지**. 착수 전 아래 "체크리스트"부터 훑는다.

## ✅ 다음 프로젝트 착수 체크리스트 (요약)

- [ ] venv는 **홈(`~/.venvs/...`)**에 만들고 **터미널마다 활성화**. 클라우드 동기화 폴더 안 금지.
- [ ] 설치는 `python -m pip`(실행 인터프리터와 동일 환경)로.
- [ ] 사내 내부망 리소스(Neo4j/게이트웨이)는 **사내망 PC에서만** 검증. 외부/샌드박스 도달 불가.
- [ ] Neo4j를 **다른 프로젝트와 공유하는지** 확인(라벨/속성/인덱스 충돌). 가능하면 전용 DB/라벨로 격리.
- [ ] 정답 값이 원문에 **읽을 수 있게** 존재하는지 먼저 진단(특히 PDF 표).
- [ ] 한글 파일명(NFC/NFD), 한↔영 표기 차이(예: 듀얼/Dual) 점검.
- [ ] LLM: OpenAI 호환은 `base_url`=`/v1`까지, 커스텀 본문은 `extra_body`. thinking 모델은 빈응답 폴백.
- [ ] **빈 응답 진단은 `max_tokens`가 아니라 입력 길이부터** — 짧은 컨텍스트는 되고 긴 것만 실패하면 **입력 한도 초과**(컨텍스트 캡으로 해결).
- [ ] 컨텍스트 구성 = 검색 품질의 핵심: **원문 1차, 구조화/그래프는 보조**.
- [ ] 스크립트·런타임에 **구조화 로깅**(실행/에러 traceback + 검색·LLM 단계) 기본 탑재.

## 1. 환경 / 인프라

| 증상 | 원인 | 해결 / 재발방지 |
|---|---|---|
| `externally-managed-environment`, venv 생성 시 `Is a directory` | 클라우드 동기화 폴더는 **심볼릭 링크를 깨뜨림**; Homebrew 파이썬은 전역 설치 차단 | venv를 **홈**에 생성, 코드만 프로젝트 폴더에서 실행. 터미널마다 `source .../activate` |
| `ModuleNotFoundError`(pydantic_settings 등)가 계속 재발 | pip 설치 인터프리터 ≠ 실행 인터프리터 | **`python -m pip install`**로 동일 인터프리터에 설치. venv 활성화 확인(프롬프트 접두사) |
| 리모트 Neo4j/게이트웨이 도달 불가 | 사내 **내부망 IP**는 외부·격리 샌드박스에서 라우팅 안 됨 | 라이브 검증은 **사내망 PC**에서. 그 외는 오프라인 검증으로 대체 |
| 파일 삭제/이동 불가(`Operation not permitted`) | 동기화 마운트 제약 | 생성/덮어쓰기로 처리, 삭제는 사용자에게 안내(또는 스텁 대체) |

## 2. 데이터 / 한글 처리

| 증상 | 원인 | 해결 / 재발방지 |
|---|---|---|
| 파일 SKIP/필터가 작동 안 함 | macOS 파일명은 **NFD**, 코드 문자열은 **NFC** → `in` 불일치 | 비교 전 `unicodedata.normalize("NFC", name)` |
| 정답이 검색돼도 답이 부정확 | **PDF 표가 텍스트화되며 행·열이 뭉개짐**(값 유실) | 구조화 추출 / 표 전용 파서(pdfplumber 등). "정답이 원문에 읽히나" 선진단 |
| "듀얼"로 물으면 0건 | 코퍼스는 영문 `Dual` 표기 | **코퍼스 실제 표기 확인** 후 동의어/쿼리 확장(한↔영 별칭) |

## 3. Neo4j / 인덱스

| 증상 | 원인 | 해결 / 재발방지 |
|---|---|---|
| 벡터 검색 `dimensionality mismatch (3072 vs 1024)` | **공유 DB**에 타 프로젝트가 같은 `:Chunk/embedding`을 3072차원으로 점유 | 전용 DB/라벨로 **격리**, 또는 초기화. `CREATE ... IF NOT EXISTS`는 동일 라벨+속성이면 skip됨에 주의 |
| 인덱스 "없음"으로 검색 실패 | 인덱스 **이름 하드코딩**(환경마다 다름) | 라벨/속성 기준 **자동 탐지** |
| 생성 직후 인덱스 미탐지로 중단 | 인덱스 생성은 **비동기**(POPULATING) | `CALL db.awaitIndexes()`로 ONLINE 대기 후 검증 |
| 별도 DB 생성 불가 | **Community 에디션은 멀티 DB 미지원** | 에디션 확인(`dbms.components`), 라벨 격리/별도 인스턴스 |

## 4. 검색 품질 (★ 핵심)

| 증상 | 원인 | 해결 / 재발방지 |
|---|---|---|
| 구조화 사실을 넣었더니 **오답** | 저품질·부분적 fact가 **앞자리·권위**로 원문 뉘앙스를 가림(손실 압축·매칭 노이즈) | **원문(청크) 1차, 구조화는 보조(뒤)** + "상충 시 원문 우선". **저점수 fact 컷** |
| 벡터는 답하는데 **Hybrid는 못 찾음** | RRF 융합이 **한쪽(벡터)만 강한 정답 청크**를 컷 밖으로 밀어냄 | **top-k 여유** + **벡터 상위 N 보장** |
| 표현이 다르면 못 찾음(CONTAINS) | 정확 문자열 매칭의 한계 | 풀텍스트(cjk)+벡터 **하이브리드**, RRF 융합 |
| **핵심 원칙** | — | *정밀 검색 ≠ 정답. 정답은 **컨텍스트 구성**(무엇을·어떤 순서로·어떤 권위로)에서 완성된다.* |

## 5. LLM 연동

| 증상 | 원인 | 해결 / 재발방지 |
|---|---|---|
| `create() got unexpected keyword 'chat_template_kwargs'` | `model_kwargs`는 create 인자로 그대로 전달됨 | 임의 본문 필드는 **`extra_body`**로 감싸기 |
| 엔드포인트 404/경로 오류 | `base_url`에 `/chat/completions`까지 포함 | OpenAI 호환은 **`/v1`까지만** |
| **답변이 빈 문자열(chars=0)** — 초기 가설 | thinking 추론 토큰이 출력 한도 소진 의심 | `max_tokens` 여유(8192) + thinking-off 재시도 + 빈응답 폴백 (※ 아래로 판명) |
| **긴 컨텍스트에서만 빈 응답** (짧으면 정상) | **입력 컨텍스트 길이 한도 초과** — 사내 모델은 입력이 일정 길이(관찰: ~6.9k OK / ~8.2k 빈응답)를 넘으면 빈 응답. `max_tokens`(출력)·thinking on/off 무관 | **입력 컨텍스트 캡**: 청크당 길이 축소(1500→900) + 총량 상한(~6000자). `max_tokens` 상향은 무효(창=입력+출력이면 오히려 악화) |
| 배치 추출이 너무 느림 | thinking 모델은 청크마다 추론 | 배치엔 **비추론 모델**(Qwen), 대화엔 thinking |
| 데모 중 RateLimit | 요청 몰림 | 에러 **상태별 처리**(재시도/다른 모델) + 폴백 |
| 답변이 너무 짧다/길다 | **답변 길이·톤은 프롬프트 지시어가 좌우**(컨텍스트 캡·검색과 무관). "간결하게"=짧게, "자세히"=길게 | 프롬프트 문구로 조절. 사용자마다 니즈 달라 **답변 스타일 옵션(간단/기본/자세히)**을 UI 선택으로 제공 |
| 나열식·반말 톤 | 프롬프트에 톤 미지정 | **상담사 톤 예시**를 프롬프트에 명시(예: "2~8등급 차등 적용" → "…차등 적용됩니다") |

## 6. 개발 프로세스 / 관측성

| 증상 | 원인 | 해결 / 재발방지 |
|---|---|---|
| 원인 추적이 오래 걸림 | 로그 부재 | **구조화 로깅 기본 탑재**: 스크립트 실행/에러 traceback + 런타임 `RETRIEVE/CTX_CHUNK/LLM_CALL/LLM_ANSWER` |
| 프론트 코드 수정이 화면에 반영 안 됨 | 컴파일된 **`.js`가 `.tsx`를 가림**(vite resolve 순서) | 소스 폴더의 산출물 `.js` 제거 + `.gitignore` |
| 네트워크 없이 검증 곤란 | 라이브 의존 | **오프라인 검증**: 구문/타입(`tsc --noEmit`)/모의 단위테스트로 로직 최대한 확인 |
| 데이터 오염/실수 복구 어려움 | 재현 절차 부재 | **재적재/초기화 스크립트**로 언제든 복구(멱등) |

| "가능?" 질문에 사실만 답하고 가부는 안 함 | 판정 로직 부재 | 임계값 테이블 + 사용자 값 대조 → **가능/불가/확인필요 판정** |

| **핵심 원칙** | — | *정밀값(금리·등급·판정)은 LLM이 아니라 **결정적 테이블**로. LLM은 서술형에 집중.* |
